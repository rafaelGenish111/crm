# הוראות שימוש בסקריפטי Seed

## סקריפטים זמינים

### 1. יצירת מנהל ראשי
```bash
npm run seed:admin
```
יוצר משתמש מנהל עם:
- אימייל: `admin@crm.com`
- סיסמה: `admin123`
- תפקיד: `super_admin`

### 2. יצירת נתוני דמו (משתתפים, קורסים, סדנאות)
```bash
npm run seed:data
```
יוצר:
- 3 מנחים (instructors)
- 8 משתתפים (students/customers) עם סיסמאות ראשוניות
- 3 קורסים עם סילבוס מלא
- 4 סדנאות
- הרשמות לקורסים וסדנאות
- מבחנים וציונים

### 3. יצירת נתוני דמו עם ניקוי נתונים קיימים
```bash
npm run seed:data:clear
```
מבצע את אותו הדבר כמו `seed:data` אבל קודם מוחק את כל הנתונים הקיימים.

- **סיסמה ראשונית** - מודפסת בקונסול בעת יצירה

## שימוש ב-Student Portal

1. הרץ את הסקריפט:
   ```bash
   cd server
   npm run seed:data
   ```

2. שמור את הסיסמאות הראשוניות שהודפסו

3. פתח את האפליקציה:
   ```bash
   npm run dev
   ```

4. גש ל-Student Portal:
   - פתח דפדפן: `http://localhost:5173/student/login`
   - התחבר עם אימייל/טלפון וסיסמה ראשונית

## מבנה הנתונים

### קורסים:
- **מבוא למדעי המחשב** - 12 מפגשים, יום ראשון
- **פיתוח Full Stack** - 16 מפגשים, יום שלישי
- **אבטחת מידע** - 14 מפגשים, יום רביעי

### סדנאות:
- סדנת Git ו-GitHub
- סדנת Docker למתחילים
- סדנת אבטחת Web Applications
- סדנת React Hooks מתקדמים

### מבחנים וציונים:
- כל קורס כולל 6 מבחנים (2 מבחנים, 2 בוחנים, 2 מטלות)
- ציונים נוצרים אוטומטית ל-50% מהמבחנים
- ציונים בטווח 60-100%

## הערות

- הסקריפט לא ימחק נתונים קיימים אלא אם משתמשים ב-`--clear`
- אם משתתף כבר קיים, הוא לא יווצר מחדש
- הסיסמאות הראשוניות מוצגות בקונסול - שמור אותן!
- כל המשתתפים נרשמים ל-2 קורסים לפחות
- כל משתתף נרשם ל-2-3 סדנאות
